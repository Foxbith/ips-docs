
# Test Case Specification

## 1. Introduction
This document provides the test cases for the Retouch Ayutthaya Affiliate Program. The test cases are designed to verify the functional and non-functional requirements outlined in the Business Requirement Document (BRD).

## 2. Test Cases

| Test Case ID | Requirement ID | Test Title/Objective | Preconditions | Test Steps | Test Data | Expected Result |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **TC-001** | FR-001 | Verify successful Referrer registration with valid data | User is on the Referrer Dashboard registration page. | 1. Fill in all required fields with valid data. <br> 2. Submit the registration form. | Individual Account Type, Valid Name, Valid ID, Valid Address, Valid Bank Info, Valid Email, Strong Password | User is registered successfully. A verification email is sent. |
| **TC-002** | FR-001 | Verify Referrer registration with invalid email format | User is on the Referrer Dashboard registration page. | 1. Fill in all required fields but use an invalid email format. <br> 2. Submit the registration form. | Invalid Email: "test.com" | An error message is displayed indicating the email format is incorrect. Registration is blocked. |
| **TC-003** | FR-001 | Verify Referrer registration with an existing email | User is on the Referrer Dashboard registration page. | 1. Fill in all required fields using an email that is already registered. <br> 2. Submit the registration form. | Existing Email: `referrer@example.com` | An error message is displayed indicating the email is already in use. Registration is blocked. |
| **TC-004** | FR-003 | Verify that Referrer can't be paid without complete personal info | Referrer is logged in but has not filled out personal info. | 1. A purchase is made by a Referee. <br> 2. Admin attempts to process payment. | Incomplete personal info | Payment is blocked. The system should indicate that personal info is missing. |
| **TC-005** | FR-004 | Verify Admin manual approval of a Referrer account | A new Referrer has registered. | 1. Admin logs into the (currently manual) backend. <br> 2. Admin verifies the new Referrer's details. <br> 3. Admin approves the account. | New Referrer account | The Referrer's account status is changed to 'Verified'. The Referrer receives full access. |
| **TC-006** | FR-005 | Verify successful Referee tracking with a referral link | App is not installed. | 1. Click on a Referrer's link. <br> 2. Install the app from the App Store/Play Store. <br> 3. Open the app. | Referral Link | The app's welcome screen shows the Referrer's name. The Referee is correctly associated with the Referrer in the backend. |
| **TC-007** | FR-006 | Verify Referee receives a discount | Referee has been tracked to a Referrer. | 1. Proceed to the in-app purchase screen. | Tracked Referee | The paywall shows the original price crossed out and a discounted price. |
| **TC-008** | FR-007 | Verify iOS "Paste" button functionality | A referral link is in the clipboard. The app is installed. | 1. Open the app. <br> 2. Tap the "Paste" button on the welcome screen. | Referral link in clipboard | The app recognizes the referral link, and the welcome screen shows the Referrer's name. |
| **TC-009** | FR-015 | Verify delayed QR code/link message | A new Referrer has just signed up and is logged in. | 1. Navigate to the main dashboard. | New Referrer account | The dashboard displays a message like "QR code and link are being generated. They will be available within 24 hours." |
| **TC-010** | FR-016 | Verify dashboard performance metrics | Referrer is logged in. | 1. Navigate to the main dashboard. | Referrer with sales data | The dashboard correctly displays Revenue, Clicks, Commissions, and Sales, and shows a graph. |
| **TC-011** | FR-020 | Verify "Info is Incorrect" status display | Admin has marked a Referrer's info as incorrect. | 1. Referrer logs into the dashboard. | Referrer account marked as 'Incorrect' | The dashboard displays a prominent message: "! Info is Incorrect (Please Edit Info)". |
| **TC-012** | FR-020 | Verify "Under Verification" status display | Referrer has submitted personal info for the first time. | 1. Referrer logs into the dashboard. | Referrer info submitted | The dashboard displays a message: "Under Verification (Within 48 hrs.)". |
| **TC-013** | NFR-005 | Verify language switching | User is on the Referrer Dashboard. | 1. Click the language toggle button. | - | The dashboard language switches between English and Thai. The setting is saved for the user. |
| **TC-014** | FR-003 | Verify validation on personal information fields | Referrer is on the 'Edit Personal Info' page. | 1. Enter an invalid ID card number. <br> 2. Enter an invalid Bank Account Number. <br> 3. Try to save. | Invalid ID, Invalid Bank Account | Error messages are shown for each invalid field. The form cannot be saved. |
| **TC-015** | FR-005 | Verify Referee tracking with QR Code | App is not installed. | 1. Scan a Referrer's QR code. <br> 2. Install the app from the App Store/Play Store. <br> 3. Open the app. | Referral QR Code | The app's welcome screen shows the Referrer's name. The Referee is correctly associated with the Referrer in the backend. |
| **TC-016** | FR-010 | Verify payout threshold | Referrer has commissions below the threshold. | 1. End of the month payout cycle arrives. | Unpaid commissions < threshold | The commission is not paid out and remains as 'Unpaid'. |
| **TC-017** | FR-010 | Verify successful payout when threshold is met | Referrer has commissions above the threshold. | 1. End of the month payout cycle arrives. <br> 2. Admin processes the payment. | Unpaid commissions > threshold | The commission is marked as 'Paid' and moved to payment history. |
| **TC-018** | FR-014 | Verify login with invalid credentials | User is on the login page. | 1. Enter an incorrect email or password. <br> 2. Click Login. | Invalid email/password | An error message is displayed. Access is denied. |
